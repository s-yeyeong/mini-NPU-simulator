# mini-NPU-simulator# My Project
# Mini NPU 시뮬레이터 프로젝트 결과 리포트

## 1. 실행 방법
* **환경:** Python 3.8 이상
* **실행:** 터미널에서 `python main.py`를 실행합니다.
* **모드 선택:** * `1`번을 누르면 3x3 필터 및 패턴을 직접 콘솔에서 입력하여 테스트할 수 있습니다. (입력 시 띄어쓰기로 구분)
  * `2`번을 누르면 동일 디렉토리에 있는 `data.json` 파일을 로드하여 5x5, 13x13, 25x25 크기의 데이터를 일괄 분석합니다.

## 2. 구현 요약
1. **라벨 정규화:** JSON 파일마다 다르게 표기된 `+`, `cross`, `x` 등의 정답(expected) 라벨을 `normalize_label` 함수를 통해 시스템 표준인 `Cross`와 `X`로 일관되게 치환했습니다.
2. **MAC 연산:** 파이썬 표준 라이브러리만 사용하였으며, 중첩 반복문(`for`)을 이용해 패턴과 필터 행렬의 원소별 곱셈 및 누적합을 직접 구현했습니다.
3. **판정 및 오차 해결 (Epsilon 정책):** 부동소수점 연산에서 발생하는 미세한 오차로 인해 잘못된 판정이 나오는 것을 방지하기 위해 `1e-9` 크기의 허용 오차(Epsilon)를 적용했습니다. 두 점수의 차이가 이보다 작으면 `UNDECIDED(판정 불가)`로 안전하게 처리하도록 설계했습니다.

## 3. 결과 리포트 및 분석

### 3-1. 실패(FAIL) 원인 분석 (0 FAIL 달성)
현재 `data.json` 분석 모드 실행 결과 **0개의 실패(FAIL)**를 기록했습니다. 이는 다음과 같은 오류 방지 로직이 성공적으로 작동했음을 의미합니다.
* **데이터 불일치 방어:** 입력값에 `+` 기호가 들어오더라도 정규화 로직을 통해 `Cross`로 완벽히 매핑되어 오답 처리를 방지했습니다.
* **동점 처리 방어:** 부동소수점 오차로 인해 `UNDECIDED`가 발생할 수 있는 아슬아슬한 케이스들을 epsilon 정책으로 걸러내어 잘못된 판정을 내리지 않았습니다.

### 3-2. 성능 및 시간 복잡도 분석 ($O(N^2)$ 증명)
아래는 모드 2 실행 시 측정된 크기별 MAC 연산 평균 시간입니다 (I/O 시간 제외, 순수 연산 10회 평균).

| 데이터 크기 (N×N) | 연산 횟수 (N²) | 평균 시간 (ms) |
| :--- | :--- | :--- |
| 5×5 | 25 | 약 0.003 ms |
| 13×13 | 169 | 약 0.015 ms |
| 25×25 | 625 | 약 0.055 ms |

**[분석 결과]**
MAC 연산은 $N \times N$ 크기의 2차원 배열을 모두 순회하며 곱셈과 덧셈을 수행하므로, 이론적인 시간 복잡도는 **$O(N^2)$**입니다. 
위 실측 데이터를 보면 데이터 크기($N$)가 5에서 25로 **5배** 커질 때, 연산 횟수($N^2$)는 25에서 625로 **25배** 증가했습니다. 평균 시간 역시 이와 비슷한 비율로 기하급수적으로 증가하는 양상을 보입니다. 
이는 복잡한 병렬 연산을 CPU 혼자 처리할 때 발생하는 병목 현상을 명확히 보여주는 결과이며, 고화질 이미지 등 대규모 데이터를 처리하는 AI 시스템에 왜 **NPU(Neural Processing Unit)와 같은 전용 하드웨어 가속기**가 필수적인지 증명합니다.
